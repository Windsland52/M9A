# 物品数据协议

> [!TIP]
>
> 本文档说明游戏中各类物品数据的存储格式和结构

## 资源存放位置

- SOS 肉鸽物品：`assets/resource/data/sos/items.json`
- 战斗物品：`assets/resource/data/combat/items.json`

## SOS 肉鸽物品数据

### 文件结构

```jsonc
{
    "artefacts": {      // 遗器数据
        "属性类型": []  // 按属性分类的遗器列表
    },
    "harmonics": {      // 和弦数据
        "属性类型": []  // 按属性分类的和弦列表
    }
}
```

### 属性类型

游戏中的五种属性类型：

- `力量` - 力量属性
- `反应` - 反应属性
- `奥秘` - 奥秘属性
- `感知` - 感知属性
- `激情` - 激情属性

### 遗器 (artefacts)

遗器按属性分类存储，每个属性包含该属性相关的所有遗器名称列表。

```jsonc
{
    "artefacts": {
        "力量": [
            "音叉残像Ⅰ",
            "音叉残像Ⅱ",
            "光荣信符",
            "力量护符",
            // ... 更多遗器
        ],
        "反应": [
            "噬人号角Ⅰ",
            "分水铜鱼",
            // ... 更多遗器
        ]
        // ... 其他属性
    }
}
```

### 和弦 (harmonics)

和弦同样按属性分类存储，每个属性包含该属性相关的所有和弦名称列表。

```jsonc
{
    "harmonics": {
        "力量": [
            "逆转之血",
            "薪血涓流",
            "血脉相承",
            // ... 更多和弦
        ],
        "反应": [
            "筹备",
            "意外觉察",
            // ... 更多和弦
        ]
        // ... 其他属性
    }
}
```

### 使用说明

这些数据主要用于：

1. **物品识别**：在 OCR 识别物品名称时作为参考字典
2. **物品选择**：在自动选择物品时根据属性优先级进行匹配
3. **数据验证**：验证识别结果是否为有效的物品名称
